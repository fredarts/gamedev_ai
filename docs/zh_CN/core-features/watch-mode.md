# 观察模式 (自动调试的魔力)

作为一名游戏开发人员 (Gamedev)，您将 **30% 的时间用于开发精彩的游戏逻辑**，而将痛苦的 **70% 的时间用于解读 Godot `Debugger / Output` 标签中的红色语法错误和空引用异常**。

Gamedev AI 的 **Watch Mode** 功能消除了这一繁琐的手动调试第二阶段。它将机器人变成了一名在隔壁房间为您工作的隐形 QA 工程师。

## 为什么要激活以及如何激活？
在聊天面板底部，激活名为 **[x] Watch Mode** 的开关。

后台发生的事情：
1. 您在 Godot 中按 `F5`（运行场景 / 调试项目）。
2. 您的玩家与一个您忘记初始化 `NodeGroup` 的 `Area2D` 发生碰撞。引擎崩溃并在输出中给出致命的红色文字 ("Attempt to call function 'die' in base 'null instance'")。
3. 无需点击或复制文本，Gamedev AI _被唤醒_，读取控制台最后 50 个块中的语法错误日志，收集可疑的索引文件，制定引用失败的假设，并立即发回 **差异重写解决方案**，或分步解释出问题的节点在哪里。

## “门槛”限制和冷却时间

过度活跃的助手会由于发送 Godot 引擎重复的 "Orphan Warnings"（实际上没有破坏任何东西的微不足道的黄色警告）而烧掉付费模型账户。

为了管理您的钱包和工作流程，Gamedev AI 通过 **动态冷却时间** 限制监控：
* 当日志在 30 秒片段中报告前 **三次** 关键输出浪潮时，它会被触发，或者有时，它会在 `standby` 中等待更长时间。

*通过阅读自主建议，照顾引擎几乎变成了小菜一碟。*
