# Пользовательский системный промпт (Custom System Prompt)

Иногда у студийных проектов есть строгие правила, и мы не хотим, чтобы ИИ принимал общие решения для всего генерируемого кода. Как вы можете сформировать дефолтное «ментальное» поведение Gamedev AI?

## Панель "Custom Instructions"

Она находится на скрытой вкладке **Settings**, под полем провайдера "API Key".
Там находится большое текстовое поле свободного ввода под названием **Custom Instructions (appended to system prompt)**.

![Панель настроек, показывающая область пользовательских инструкций](../../images/settings_ui.png)

Когда ИИ «думает», сохраненные там инструкции читаются как глобальное правило №1 перед тем, как ответить вам.

### Примеры содержания этого поля:
* *"Пожалуйста, не пиши длинных объяснений или приветствий, просто показывай сфокусированное окно диффа (Diff)"*
* *"Все проекты Godot в этой студии используют соглашения Clean Code. Методы должны начинаться с маленькой английской буквы, а узлы сцены должны быть в PascalCase. Строго используй статическую типизацию (Static Typing) во всех создаваемых функциях (: String, : int, -> void)."*
* *"Я запрещаю использовать _process(). Я хочу, чтобы ты сосредоточился на архитектуре с интенсивным использованием сигналов (Signals) для производительности."*

Благодаря этому Gamedev AI становится разработчиком, специально обученным техническим директором (вами), и создает код в том стиле, который одобряет ваша команда, вместо того чтобы следовать общему синтаксису LLM.
